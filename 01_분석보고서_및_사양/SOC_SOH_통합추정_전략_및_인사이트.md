# SOC 및 SOH 동시(통합) 추정 알고리즘 개발 전략 및 기법 비교

## 1. 개발 전략 로드맵 (Strategy Pipeline)
새로운 초고성능 통합 알고리즘을 만들기 위해서는 단계적이고 정량적인 접근이 필요합니다.

- **Phase 1 (Baseline 구축):** 현재 보유하신 데이터(`Cell 데이터★★★★★`)를 기반으로 전통적인 SOC와 SOH 단일 추정 모델(EKF, LSTM 등)을 각각 구현하여 오차율(RMSE, MAE) 기준점(Baseline)을 잡습니다.
- **Phase 2 (통합 모델 설계):** 아래 "인사이트" 섹션에서 제안하는 하이브리드 또는 듀얼 아키텍처 중 하나를 채택하여 "하나의 알고리즘"으로 병합합니다.
- **Phase 3 (학습 및 튜닝):** 입력 변수(V, I, T 등)를 모델에 넣고, SOC 추정 오차와 SOH(단기 저항 R0 변화, 장기 용량 감소) 오차의 Loss를 동시에 최소화하도록 멀티태스크(Multi-task) 학습/옵티마이징을 수행합니다.
- **Phase 4 (비교 평가):** Phase 1의 단일 모델들의 결과와 Phase 3의 통합 모델 결과를 시각화(Error Graph)하여 우수성을 증명합니다.

## 2. 기존 SOC, SOH 주요 기법 비교 평가표

### 2.1. SOC 추정 기법
| 기법 (Method) | 장점 (Pros) | 단점 (Cons) | 요약 |
|:---|:---|:---|:---|
| **Ah Counting** (전류적산) | 직관적이고 연산량이 매우 낮음 | 초기 오차 및 렌서 노이즈가 시간이 지날수록 **무한대로 누적됨** | 단독 사용 불가 (타 기법과 병행 필수) |
| **OCV Look-up** | 매우 높은 정확도 (배터리 고유 특성) | 휴지 시간(Relaxation)이 길게 필요하여 **동적(실시간) 환경에서 사용 불가** | OCV-SOC 맵퍼(Mapper)로서 보정용으로 쓰임 |
| **Kalman Filter** (EKF/UKF/CKF) | 노이즈 환경에 강건하며 실시간 동기화 우수 | 내부 등가회로모델(ECM) 파라미터가 노화(SOH)되면 부정확해짐 | SOH 변환 없이 쓰면 시간에 따라 성능이 계속 떨어짐 |
| **Data-Driven** (LSTM, CNN) | 비선형적 배터리 화학 특성 학습이 통째로 가능 | 막대한 데이터 소모, BMS 칩셋 연산량 한계 존재 | 데이터 퀄리티(Data Quality)에 결과가 100% 좌우됨 |

### 2.2. SOH 추정 기법
| 기법 (Method) | 장점 (Pros) | 단점 (Cons) | 요약 |
|:---|:---|:---|:---|
| **Direct Measurement** | 가장 정확 (완충~완방 용량 측정) | 전기차나 ESS 실사용 중에 완충/완방을 강제할 수 없음 | 오프라인 RPT(실험실) 용도로만 한정됨 |
| **ECM 저항 기반 (R증가)** | 물리적 의미 부여가 직관적임 | 동적 주행 상황에서 R0, R1 등을 분리 추출하기 매우 까다로움 | 필터링 기법과 엮어 파라미터 추출(Identification) 필요 |
| **HI (Health Indicator) 기반** | 전압/전류 커브의 특정 구간(피크점, 상승시간 등)만 보고 모델링 | 다양한 주행 패턴에서는 해당 특정 커브(HI) 점을 잡기가 어려움 | 고정된 충전 프로파일(CC-CV) 시점에 주로 사용됨 |

## 3. [인사이트] 초고성능 '통합(Joint)' 추정 알고리즘 아이디어

배터리 연구의 최신 트렌드를 볼 때, SOC와 SOH는 뗄 수 없는 관계(Coupled)입니다. SOH가 변하면 전체 용량이 변해 SOC 계산이 틀려지고, SOC가 틀려지면 특정 SOC 구간의 OCV가 어긋나 SOH 저항 예측이 박살납니다. 서로 상호보완적으로 "동시에" 풀면 단독 기법들보다 훨씬 우수한 성능을 낼 수 있습니다. 

추천 방식은 크게 세 가지 형태가 있습니다.

### 인사이트 1: 이중 칼만 필터 (Dual EKF 또는 Joint EKF)
- **개념**: 필터를 두 개 직렬/병렬로 꼬아서 만듭니다. 첫 번째 필터는 전압/전류로 **SOC**를 실시간으로 쫓아가고, 두 번째 필터는 첫 번째 결과를 이어받아 배터리의 파라미터(내부저항 R, 정전용량 C)를 천천히 업데이트하여 **SOH**를 업데이트합니다. 그 SOH 결과를 다시 첫 번째 필터에 피드백합니다.
- **장점**: 수많은 논문에서 입증된 안정성. 메모리가 적어도 동작 가능(임베디드 BMS 탑재 친화적).
- **차별화 포인트(우수성)**: 노이즈 공분산(Q, R 매트릭스)을 고정하지 않고, 파이프라인 안에서 머신러닝이나 퍼지 로직(Fuzzy Logic)으로 "적응형(Adaptive)"으로 조정하게 만들면(AEKF 기반) 기존 수식 기반 필터들보다 훨씬 압도적인 성능을 뽑아낼 수 있습니다.

### 인사이트 2: 멀티-태스크 딥러닝 (Multi-Task TCN / Transformer)
- **개념**: 과거 N초간의 V, I, T 시계열 데이터를 딥러닝 모델에 통과시킵니다. 인코더(Encoder)에서 배터리 상태 표상(Feature Representation)을 하나로 깊게 뽑은 뒤, 아웃풋(Head)을 스플릿하여 하나는 단기 활성 상태인 **SOC**를 출력하고, 하나는 장기 퇴화 상태인 **SOH(또는 저항치)**를 출력합니다.
- **장점**: 서로 연관된 두 과제를 동시에 학습(Multi-task Learning)하므로, 모델이 배터리 내부 동역학을 훨씬 깊이 이해하게 되어 오차(Overfitting)가 대폭 줄어듭니다.
- **차별화 포인트(우수성)**: 기존 AI 연구들은 LSTM 기반이 많은데, TCN(Temporal Convolution Network)이나 Attention 기반 인코더를 쓰면 연산은 더 빠르면서 기존의 낡은 모델들을 성능 면에서 모두 꺾을 수 있습니다. 보유 중이신 에기연 `Pattern` 폴더(동적 데이터)를 활용하기 아주 완벽합니다.

### 인사이트 3: 물리 지식 기반 인공신경망 (Physics-Informed Neural Network, PINN) 🏆 초강력 추천
- **개념**: 1번(모델 기반)과 2번(데이터 기반)의 완벽한 융합입니다. 딥러닝을 하지만, 뉴럴네트워크의 Loss Function에 **배터리의 OCV-SOC 방정식, 등가회로모델의 물리적 수식 차이값(Residual)**을 벌점(Penalty)으로 집어넣습니다.
- **장점**: 블랙박스인 AI에 "물리법칙"을 가르치는 방법입니다. 학습하지 않은 기괴한 주행 패턴이 들어와도 물리 수식이 통제하므로 SOC나 SOH가 절대 터무니없는 값으로 튀지 않습니다. (강건함 압도적 개선)
- **차별화 포인트(우수성)**: 논문이나 프로젝트 평가에서 가장 높은 점수(혁신성)를 받을 수 있으며, 기존 어떤 전통적 SOC, SOH 단일 기법들보다 월등한 오차율(특히 수명이 다 되어갈 때 RMSE 성능)을 기록합니다.

## 4. 넥스트 스텝
이 중에서 기술적으로나 실험적으로 가장 끌리시는 방향성(예: **"나는 딥러닝 쪽으로 파볼래!"** 또는 **"나는 코딩 최적화를 곁들인 필터(EKF) 쪽으로 먼저 접근해볼래"**)이 있으신가요? 

방향성만 짚어주시면 곧바로 `Cell 데이터★★★★★`에서 해당 알고리즘의 Input에 맞추어 데이터를 썰어내는 전처리 코딩부터 단계별로 시작하겠습니다.
